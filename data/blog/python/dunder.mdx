파이썬을 사용하다보면 `__init__`, `__str__`, `__repr__`등의 함수들을 자주 보게 된다.
이러한 함수들을 **던더 함수** 혹은 **매직 메소드**라고 부른다.
던더 함수는 파이썬의 *특별한 메소드*로, **클래스 내부에 정의**되어 **객체의 특정 동작을 정의**하는 데 사용된다. 
이러한 메소드들은 두 개의 언더스코어(`__`)로 둘러싸여 있어 "던더"라는 이름으로 불린다.

## 던더 함수의 종류
던더 함수는 다양한 종류가 있으며, 그 중 일부는 다음과 같다:
- `__init__`: 객체가 생성될 때 호출되는 초기화 메소드.
- `__str__`: 객체의 문자열 표현을 반환하는 메소드.
- `__repr__`: 객체의 공식적인 문자열 표현을 반환하는 메소드.
- `__add__`: 두 객체를 더할 때 호출되는 메소드.
- `__len__`: 객체의 길이를 반환하는 메소드.
- `__getitem__`: 객체의 인덱싱을 지원하는 메소드.
- `__setitem__`: 객체의 인덱싱을 통해 값을 설정하는 메소드.
- `__delitem__`: 객체의 인덱싱을 통해 값을 삭제하는 메소드.
- `__iter__`: 객체를 반복 가능하게 만드는 메소드.
- `__next__`: 반복자에서 다음 값을 반환하는 메소드.
- `__call__`: 객체를 함수처럼 호출할 수 있게 하는 메소드.
- `__enter__`: 컨텍스트 매니저의 시작을 정의하는 메소드.
- `__exit__`: 컨텍스트 매니저의 종료를 정의하는 메소드.
- `__eq__`: 두 객체가 같을 때 호출되는 메소드.
- `__lt__`: 두 객체가 작을 때 호출되는 메소드.
- `__gt__`: 두 객체가 클 때 호출되는 메소드.
- `__le__`: 두 객체가 작거나 같을 때 호출되는 메소드.
- `__ge__`: 두 객체가 크거나 같을 때 호출되는 메소드.
- `__ne__`: 두 객체가 다를 때 호출되는 메소드.
- `__hash__`: 객체의 해시 값을 반환하는 메소드.
- `__bool__`: 객체의 불리언 값을 반환하는 메소드.
- `__del__`: 객체가 삭제될 때 호출되는 메소드.
- `__format__`: 객체의 형식을 지정하는 메소드.
- `__getattr__`: 객체의 속성을 동적으로 가져오는 메소드.
- `__setattr__`: 객체의 속성을 동적으로 설정하는 메소드.
- `__delattr__`: 객체의 속성을 동적으로 삭제하는 메소드.
- `__dir__`: 객체의 속성과 메소드 목록을 반환하는 메소드.
- `__getstate__`: 객체의 상태를 반환하는 메소드.
- `__setstate__`: 객체의 상태를 설정하는 메소드.
- `__reduce__`: 객체의 직렬화 정보를 반환하는 메소드.
- `__reduce_ex__`: 객체의 직렬화 정보를 반환하는 메소드 (파이썬 2와 호환).
- `__getnewargs__`: 객체의 생성 인자를 반환하는 메소드.
- `__getnewargs_ex__`: 객체의 생성 인자를 반환하는 메소드 (파이썬 2와 호환).
- `__getinitargs__`: 객체의 초기화 인자를 반환하는 메소드.
- `__getinitargs_ex__`: 객체의 초기화 인자를 반환하는 메소드 (파이썬 2와 호환).

`self`는 던더 함수의 첫 번째 인자로 항상 사용되는 특별한 변수이다.
`self`는 클래스의 인스턴스를 참조하는 데 사용되며, 던더 함수 내에서 인스턴스의 변수(instance variable)이나 메소드(instance method)에 접근할 수 있게 해준다.
